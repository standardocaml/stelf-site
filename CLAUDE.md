# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — start the local dev server at `localhost:4321`
- `npm run build` — build the production site to `./dist/`
- `npm run preview` — preview the built site locally
- `npm run astro -- check` — run Astro's type/content checks (closest thing to a lint/test pass)
- `make build` / `make dev` — Makefile wrappers for the above

There are no dedicated lint or test scripts in `package.json`.

## Architecture

This is an **Astro + Starlight** documentation site for STELF (a variant/port of Twelf).

- `astro.config.mjs` — central config: sidebar structure, math plugins (`remark-math`, `rehype-mathjax`), and MDX. **Adding a new guide section requires updating the sidebar here.**
- `src/content.config.ts` — registers the `docs` collection using Starlight's `docsLoader()` and `docsSchema()`.
- `src/content/docs/` — all documentation pages; file paths map directly to URL routes.
  - `guides/ProvingMetaTheorems/` — multi-part tutorial (ported from "Proving Metatheorems with Twelf"), split across `FirstOrder/`, `HigherOrder/`, `Advanced/` subdirectories.
  - `reference/` — reference pages (Grammar, Config, Modules, Pal, etc.); sidebar is **autogenerated** from this directory.
  - `reference/Archive/` — legacy/archived content; still follows normal Starlight rules.
- `src/components/` — custom Astro components imported in MDX via the `custom/` alias:
  - `BNF.astro` — renders BNF grammar notation with `kind` prop (`rep1`, `rep0`, `alt`, `opt`).
  - `Math.astro` — inline/block math wrapper.
  - `Todo.astro` — renders a Starlight caution badge marking incomplete sections.
- `twelf/` — large collection of `.elf` source files (Twelf/STELF examples and archive content); these are reference material, not built by Astro.

## Content conventions

- Pages use MDX (`.mdx`); frontmatter requires at minimum a `title` field.
- Import custom components with `import BNF from "custom/BNF.astro"` etc.
- Math is written as fenced code blocks with `math` language tag or via `<Math>` component; MathJax renders it.
- The `reference/` sidebar is autogenerated — file names become nav labels; no manual sidebar entry needed for new reference pages.
- Guides must be manually added to the `sidebar` array in `astro.config.mjs`.
- Renaming a file changes its URL; internal `/wiki/...` links in content refer to routes by slug.
